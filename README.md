# NewsPrism

ニュースを **3つの軸（経済・社会・外交安保）** で分析し、背景思想を理解しながら記事を読むツールです。

---

## できること

### ニュースフィード
複数のメディアのニュースをトピック別（外交・政治・経済など）のカラムで表示します。気になる記事をすぐに分析できます。

### 3軸分析
記事を読んで「この記事はどんな立場で書かれているか」を3つの軸でスコア化します。

| 軸 | 左（-1.0） | 右（+1.0） |
|---|---|---|
| 経済 | 市場重視・小さな政府 | 再分配・社会保障重視 |
| 社会 | 伝統・秩序重視 | 多様性・個人の自由重視 |
| 外交安保 | 抑止力・タカ派 | 対話・ハト派 |

- 感情トーン（煽り度）の検出
- あえて逆の立場からの反論を自動生成
- 複数記事をプロットで一覧比較

### ニュースまとめ（/ranking）
大手メディア15社のRSSを自動収集し、同一ニュースイベントをグループ化してランキング形式で表示します。複数媒体が報じているトピックを上位に表示し、「どのメディアが報じてどのメディアが沈黙しているか」をひと目で確認できます。

### メディア比較
同じニュースを複数のメディアがどう報じているかを並べて比較できます。「NHKと産経と朝日でこの事件をどう伝えているか」がひと目でわかります。

### グループ点検（/inspect）
グループ化ロジックの動作を確認できる開発者向けツール。

| タブ | 内容 |
|---|---|
| FeedGroups（DB） | インクリメンタルグループの一覧。多媒体 / 単独報道の内訳を表示 |
| Snapshot（バッチ結果） | 最新バッチ実行結果。報道媒体・沈黙媒体を確認できる |

グループを展開すると自動検出された品質 issue が表示されます（高: カテゴリ混在 / 中: サブカテゴリ混在 / 低: カテゴリ未設定）。

### バッチの動作モード

| 環境変数 | 効果 |
|---|---|
| `SKIP_REFINE=true` | refine ステップ（クラスタ品質審査）をスキップ。LLM 負荷を下げたい場合や高速実行したい場合に使用 |

## アーキテクチャ

本プロジェクトは **Go (Backend)** と **Next.js (Frontend)** のハイブリッド構成です。

- **shared (Go)**: DBアクセス、LLM連携、共通設定などの共有モジュール。
- **batch (Go)**: RSS収集、LLM分析、グループ化、DB保存を行うバックグラウンドプロセス。
- **server (Go)**: フロントエンド向けの API サーバー（Next.js API Routes を順次移行中）。
- **src (Next.js)**: React Server Components を活用したフロントエンド。

## 使い方

**記事を分析する**
1. ニュースフィードから気になる記事の「3軸分析」をクリック
2. または上部の入力欄に記事のURLを貼り付けて「分析」

**メディアを比較する**
1. 「メディア比較」ページを開く
2. キーワードを入力（例: 防衛費、少子化対策）
3. 同一ニュースのグループが表示されたら、比較したいグループを選択

**フィードをカスタマイズする**
- ⚙ ボタンでフィード設定を開き、表示するメディアやトピックを変更できます
- 設定はブラウザに保存されます
