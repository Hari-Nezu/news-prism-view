# 報道記事・動画ニュースの権利 徹底調査

> NewsPrism のマネタイズに先立ち、報道コンテンツの権利処理について「安全 / グレー / 危険」を分類整理。
> 結論先出し：**「3軸スコア + 短い要約 + 元記事へのリンク」+「RSS / 公式 API 経由のみ」** が最も安全。記事本文の保存・全文要約・サムネ転載・動画埋め込みの常用は地雷。
>
> 調査日: 2026-06-09 / 対象法域: 主に日本（米国の判例動向は補足）

---

## 0. 結論サマリ（先に読む）

| 取り扱い | 安全度 | NewsPrism の現設計との関係 |
|---|---|---|
| **見出し + URL + 公式RSSの範囲** | ◎ 安全 | 現状この方針。維持すべき |
| **記事の3軸スコア / メタデータのみDB保存** | ◎ 安全 | 「事実・データ」は著作物でない。維持 |
| **自社で生成した短い要約（事実抽出）+ リンク** | ○ 安全（条件あり） | リード文の創作的表現を流用しないこと |
| **記事本文をDBに長期保存（社外非公開）** | △ グレー | 30条の4「情報解析目的」で正当化できる余地はあるが、利用規約違反になりうる |
| **記事本文をユーザーに再表示（全文 / 長文要約）** | × 危険 | 複製権・公衆送信権侵害の典型 |
| **見出し+リード文をそのまま表示** | × 危険 | リード文には著作物性が認められる判例あり |
| **新聞社サイトのスクレイピング（規約禁止）** | × 危険 | 利用規約違反 + 不正競争防止法・偽計業務妨害リスク |
| **テレビニュース動画のサムネ / フレーム転載** | × 危険 | 41条の「報道のための利用」は引用元側にしか適用されない |
| **YouTube公式埋め込み（iframe）** | ○ 安全 | YouTube ToS で許諾済。ただし違法アップ動画は× |
| **公式 API（NHK・時事通信・Reuters 等）契約** | ◎ 最安全 | 法人プラン化したいなら避けて通れない |

## 1. 法的フレームの整理（日本）

### 1.1 著作物性が「ない」もの

著作権法 **10条2項**：「事実の伝達にすぎない雑報及び時事の報道」は著作物に該当しない。
- 訃報、人事異動、株価、スポーツの試合結果など **事実だけ** の短報がこれ。
- **誤解しがちな点**: 通常のニュース記事は「記者の取材・選択・表現」が入るので **著作物**。10条2項に逃げ込めるのは事実列挙の短報のみ。

### 1.2 見出し（タイトル）

**読売オンライン事件（知財高判 平成17年10月6日）**：YOLの記事見出しの著作物性を否定。
- ただし判決は同時に、見出しに **「法的保護に値する利益」** を認め、**不法行為（民法709条）** での損害賠償を肯定した。
- → 見出しの **大量・組織的・営利目的の利用** は著作権侵害ではなくとも不法行為になりうる。
- NewsPrism での解釈: **見出しを単発でリンクと共に表示** は問題なし。**見出し+リード文を全社分まとめてアーカイブして検索させる** は読売オンライン事件のリスクゾーンに近い。

### 1.3 リード文・記事本文

- 通常の記事本文は **著作物**（複製権・翻案権の対象）。
- **リード文も著作物性が認められる判例多数**（記者の表現が凝縮されているため）。
- 「事実だけを抽出した要約」（=自分の言葉で書き直した事実列挙）は、翻案にあたらない＝OK。
  - 境界は曖昧。**原文表現を残す＝翻案権侵害**、**事実だけ抜く＝OK**。

### 1.4 引用（32条）

要件（パロディ・モンタージュ事件・最高裁）：
1. **公表された** 著作物であること
2. **明瞭区別性**（カギ括弧・引用ブロック等で視覚的に区別）
3. **主従関係**（自分の文章が「主」、引用が「従」）
4. **目的上正当な範囲**（必要最小限）
5. **出所明示**
6. 改変しない（同一性保持権）

NewsPrism での適用可否:
- 「3軸スコアの根拠としてA社の記事のこの一文を引用」のような **個別記事ページ** であれば引用として成立しうる。
- 「ランキング画面に各社の見出しを一覧表示」は **引用の主従関係を満たさない**（NewsPrismの記述が「主」になっていない）→ 引用ではない別の根拠が必要。

### 1.5 時事の事件の報道（41条）

「**自ら報道を行うため**」に、その事件を構成する著作物等を **報道のための正当範囲** で利用できる規定。
- 想定: 盗まれた絵画の写真を新聞が掲載、事件現場の壁画を映す等。
- **重要**: NewsPrism のような **アグリゲーター / 横断比較プラットフォーム** は、自らがオリジナル取材で「報道」をしているわけではないため、**41条で正当化するのは厳しい**。
- 「報道に関する論評」は 32条の引用で構成すべきで、41条で押し切るのは危険。

### 1.6 30条の4（情報解析のための利用）

「思想・感情の **享受を目的としない** 利用」は原則自由。LLM学習・情報解析（クラスタリング・スコアリング）はこれに該当する余地。

- **NewsPrism のバッチ処理（埋め込み・分類・3軸スコアリング）はこの枠で説明可能**。
- ただし但し書き「**著作権者の利益を不当に害する場合は除く**」。文化庁見解（2024.3 / 2025.9資料）では:
  - 有料DB・有料サイトを **回避して** スクレイピングするのは但し書きに該当しうる
  - **海賊版サイト** からのデータ収集は不可
  - **作風を模倣する目的** などは別途侵害になりうる
- スコアリング目的での **記事の中間保存（DB保存）** は 30条の4 の範囲で正当化できる可能性が高いが、その出力をユーザーに **元の表現として再表示する** 段階で享受目的に転化＝条件を外れる。

### 1.7 著作隣接権・データベース権

- **編集著作物 / データベース著作物**: 個別記事に著作権がなくても、「記事DBの選択・体系性」自体が著作物になりうる（タウンページ事件）。新聞社の記事DBを丸ごとコピーするのは別途侵害。
- **不正競争防止法 2条1項 14号**: 「営業上の利益を侵害する目的で、限定提供データを不正に取得・使用」した場合に違法。新聞各社の有料記事DBがこれに該当する余地。

## 2. パターン分類

### ✅ 安全パターン（自信を持って実装してよい）

#### A1. 公式 RSS / API 経由でメタデータ取得 + 自社解析

- **やること**: 各社の公式RSSから「タイトル / URL / 公開日時 / カテゴリ / OGP画像URL（リンク）」を取得。DBには `url, headline, source_id, published_at, scores...` のみ保存。
- **根拠**: RSS配信は事実上「指定範囲での再配信を許諾」。NewsPrism は **配信されたメタデータ + 自社生成スコア** を扱うだけで、原文の複製公衆送信は行わない。
- **注意**: RSS配信元の **利用規約を1社ずつ確認**。時事通信は「個人非営利」限定の規約あり。商用利用は **要問合せ** のところが多い。

#### A2. 3軸スコア・統計・メタデータのみの可視化

- **やること**: メディアA「経済 -0.7 / 社会 +0.3 / 外交 -0.4」のような **数値** と **集計** を表示。記事の中身は出さない（タイトル+リンクのみ）。
- **根拠**: スコアは NewsPrism が独自に生成した二次データ。事実+独自解析であり著作物性は NewsPrism 側に発生。

#### A3. 短い「事実抽出型」要約 + 元記事リンク

- **やること**: 「メディアAは○月○日に○○を報じた。論調はXX」のように **事実だけ** を自分の言葉で書き、原文の表現は使わない。
- **根拠**: 事実は著作物ではない。翻案にあたらない範囲なら 30条の4 + 表示時の事実利用で成立。
- **境界線（要注意）**: LLMで要約させると原文の言い回しが残りやすい。**「事実だけ抽出して50字以内で書き直す」** のような明示プロンプトと、コサイン類似度 / n-gram 一致での検査が必要。

#### A4. YouTube 公式埋め込み（iframe）

- **やること**: YouTube 公式ページの「共有→埋め込み」で吐き出される iframe をそのまま貼る。
- **根拠**: YouTube ToS で埋め込みが明示的に許諾されている。**ロケットニュース24事件・最判平29** など、リンク・埋め込みは公衆送信権侵害にならないとの判決傾向。
- **NG 系**:
  - 違法アップロード動画と知りつつ埋め込む（民事責任の余地）
  - サムネ画像を **iframe ではなく自前で取得して再配信** する
  - 動画をダウンロードして自社サーバから配信

#### A5. 公式ライセンス契約（最強だが高い）

- **時事通信社 / 共同通信 / Reuters / AP** などのコンテンツライセンス契約。
- 法人プラン（マネタイズ計画の階層C）を本気でやるならここに到達する必要がある。
- 個人開発フェーズでは「契約交渉に体力を使わない」のが正解。

### ⚠️ グレーパターン（条件付き・判断が分かれる）

#### G1. 記事本文を内部 DB に保存（情報解析目的）

- **状況**: スコアリングのため記事本文を一度 DB に入れる。ユーザーには表示しない。
- **法的整理**: 30条の4 の「情報解析」で正当化可能。ただし、
  - スコアリング後に削除する設計（保存期間を明示）
  - 漏洩で外部に出ない設計（アクセス制御）
  - 元サイトの **利用規約に「複製禁止」「データベース作成禁止」が明記されていないか** を確認
- **NewsPrism の現状**: バッチで取得した本文をどう扱っているかは要確認。「分析後は破棄 / 数値化のみ残す」が安全側。

#### G2. 規約に「個人利用のみ」と書かれた RSS の商用利用

- 時事通信 RSS は **個人非営利のみ**。これを商用サービスに組み込むのは規約違反。
- 法的には「契約違反 / 利用承諾の前提を満たさない」なので、 **著作権法以前に契約法レベルで NG**。
- 商用使うなら **時事通信に商用ライセンスを直接問合せ** が必要。

#### G3. 見出しの大量集約・検索可能アーカイブ

- 単発の見出し転載はOKだが、**「過去数年分の全社見出しを検索 / 比較できる」** はYOL事件の不法行為判断のリスクゾーン。
- 緩和策:
  - 見出しの **タイミング表示はリアルタイム / 直近のみ**
  - **アーカイブは「数値・スコア」のみで、見出しテキストは出さない**（クリックで元記事URLへ）

#### G4. LLM 逆張り反論生成（NewsPrism の機能）

- 30条の4 で記事を読み込んで反論生成 → これは享受目的に近いとみなされる可能性。
- 緩和策:
  - 反論文に **元記事の表現が残らない** よう生成（要約→反論ではなく、3軸スコアと話題ラベルから反論）
  - 反論文中に元記事をそのまま引用する場合は **32条の引用要件** をUI上で満たす（明瞭区別、出所明示、主従関係）

#### G5. OGP画像の自前配信（プロキシ・キャッシュ）

- 元記事の OGP 画像（多くは記者・カメラマンの著作物）を NewsPrism 側でキャッシュ配信するのは複製・公衆送信。
- 安全策: **元サーバの URL を img タグの src に直接指定**（リンクのみ）。CDN 経由のキャッシュ配信はやらない。
- パフォーマンス的に自前配信したくなるが、権利的にはやらないのが無難。

### 🚫 危険パターン（やってはいけない）

#### D1. 記事本文の全文 / 長文要約のユーザー表示

- 複製権・翻案権・公衆送信権の **直接侵害**。差止 + 損害賠償の典型。
- 「要約だから OK」は誤解。原文の **表現上の本質的特徴を直接感得できる** 要約は翻案侵害（江差追分事件・最判平13）。

#### D2. リード文をそのまま表示

- リード文は記者の表現が凝縮されており著作物性が認められやすい。
- RSS にリード文（description）が含まれている場合でも、**そのまま転載するのは別問題**。利用規約に許諾がない限り NG。

#### D3. robots.txt / 利用規約で禁止された記事サイトのスクレイピング

- **岡崎市立中央図書館事件（Librahack）**: 1秒1回の合法アクセスでも、図書館側がクラッシュしたことで **偽計業務妨害で逮捕**（不起訴）。技術的には合法でも刑事リスクが残る前例。
- **不正競争防止法 2条1項14号**: 限定提供データの不正取得。
- **私的利用の範囲を超える組織的スクレイピング → 民事 + 刑事リスク**。

#### D4. テレビニュース動画のフレーム・キャプ画像転載

- 41条「時事の事件の報道のための利用」は **NewsPrism のような横断比較サービスには適用されない**（自社で報道していないため）。
- 32条の引用で押し切るには、明瞭区別・主従・必要最小限を厳格に満たす必要があり、ランキング画面のような **一覧用途では引用要件を満たせない**。

#### D5. 違法アップロード動画の埋め込みを継続

- 違法と知りつつ埋め込むと **間接的に侵害幇助** とみなされる余地（リーチサイト規制：著作権法 113条2項以降）。
- TBS / NHK 等公式チャンネルから埋め込むのは安全。**個人アップの「ニュース切り抜き」を埋めるのは危険**。

#### D6. データセット販売としての「記事本文＋メタデータ」の販売

- マネタイズ計画の階層D「データ販売」で、**記事本文を含めて販売** するのは絶対NG。
- 販売してよいのは:
  - **NewsPrism が独自に算出したスコア・分類**
  - **URL・公開日・媒体名・公開カテゴリ** などの事実情報
  - **要約は含めない**（要約の著作物性が NewsPrism 側にあっても、原記事の翻案権を侵害するため）
- 学術用途で本文も含めたい場合は、**「研究目的・47条の5に類似する設計」+ 利用機関に元媒体との契約を求める** など段階を踏む必要あり。

#### D7. 「沈黙メディアの可視化」で記事が **無いこと** を主張する場合

- 著作権は問題にならないが、**名誉毀損 / 信用毀損** のリスクが別軸で発生。
- 「Aメディアは報じなかった」は事実主張なら問題ないが、「Aメディアは意図的に隠した」は別。
- 緩和策: **データに基づく定量表示にとどめ、意図の評価は読者に委ねる UI**（現状の3軸方針と整合）。

## 3. 動画系ニュース固有の論点

### 3.1 YouTube ニュース（公式チャンネル）

| 行為 | 可否 |
|---|---|
| 公式iframe埋め込み | ◎ ToSで許諾済 |
| サムネ画像を自前で取得して表示 | △ 規約上はOEmbed推奨 |
| 動画タイトル + URL の一覧表示 | ◎ |
| YouTube Data API でメタデータ取得 | ◎ API ToS範囲内 |
| 動画を文字起こし → DB保存（自社解析） | △ 30条の4で正当化余地。ただし YouTube ToS は「ダウンロード禁止」明記 |
| 文字起こしをユーザーに表示 | × ToS違反 + 著作権侵害 |

### 3.2 テレビニュース（地上波・BS）

- 地上波の番組は **放送事業者の著作隣接権** + 番組制作者の著作権の二重構造。
- 番組のキャプチャ画像を「報道トーン分析」に使うのは:
  - 41条 → アグリゲーターには適用困難
  - 32条 → ランキング画面では主従関係を満たせない
- **避けるのが無難**。やるなら **「該当時刻に該当チャンネルで○○分の報道があった」というメタデータのみ**（M Data 等のEPGデータライセンス）。

### 3.3 ニュースサイトの埋め込み動画（自社CMSの動画プレイヤー）

- 朝日新聞デジタル等の自社動画プレイヤーには公式の埋め込み機能が **ない場合が多い**。
- 無理に iframe で読み込むのは、相手サーバの規約違反になりうる。
- 安全策: **記事URLへのリンクのみ**。

## 4. NewsPrism への具体的提言

### 4.1 すぐにやるべきこと

1. **取得元ごとの利用規約・RSS規約を1社ずつ精査し、社内ドキュメント化**
   - `docs/source-licenses.md` を作って「媒体 / 取得手段 / 規約URL / 商用可否 / 注意点」を表で管理
2. **DBスキーマの権利監査**
   - `articles.content` 相当のフィールドがあるなら、**スコアリング後に空にする** バッチを追加
   - 残すのは `url, headline, source_id, published_at, scores, embeddings, topic_id` まで
3. **UI から「リード文の長文表示」を排除**
   - ランキング画面・記事一覧で表示するのは **タイトル（リンク）+ 自社生成の事実要約（50字以内）+ 3軸スコア** まで
4. **YouTube 統合は公式 iframe + Data API のみ**
   - 「カスタムソース追加」機能で個人アップ動画を追加させる場合は **公式チャンネル判定** または **責任は利用者にある旨の同意** を取る

### 4.2 マネタイズ計画への影響

| マネタイズ計画 | リスク評価 |
|---|---|
| 階層A: 広告（AdSense） | ◎ コンテンツが現方針なら問題なし |
| 階層B: Pro サブスク | ○ 「過去アーカイブ全期間検索」は **スコア・メタデータ検索** に限定すれば安全。記事本文検索はNG |
| 階層C: 法人ライト | △ 「報道トーン監視」は安全だが、**法人にレポートを納品する** 段階で「要約に原文表現が混入」のチェックが必須。納品物は **数値+グラフ+リンク** に絞る |
| 階層D: データ販売 / API | ⚠️ **記事本文・要約は売らない**。スコア・URL・媒体名・タイムスタンプのみ。学術提供は別途相談スキーム |

### 4.3 法務的に「武器化」できる差別化

逆に、権利処理を **真面目にやっている事業者** は競合のニュースまとめサイトより圧倒的に信頼される。

- **「全データはRSS/公式API由来。原文の保存も再配信もしない」** をプライバシーポリシー兼ライセンス方針として掲示
- **「3軸スコア算出ロジック」のホワイトペーパー公開** で透明性アピール（マネタイズ計画の「採点ロジック公開」と整合）
- 法人向け資料に「権利処理ポリシー」を1ページ入れる → 広報・PR部門の購買決裁が通りやすくなる
